<feature>
  <name>Session name sanitization and remote command builder</name>
  <files>src/session.rs, src/lib.rs</files>
  <behavior>
    sanitize_session_name:
    - "dev.frontend" -> "dev-frontend" (dots to hyphens)
    - "servers:prod" -> "servers-prod" (colons to hyphens)
    - "my.workspace:test!" -> "my-workspace-test" (special chars stripped, trailing hyphens trimmed)
    - "simple" -> "simple" (alphanumeric unchanged)
    - "a_b-c" -> "a_b-c" (underscores and hyphens preserved)
    - "--leading" -> "leading" (leading hyphens trimmed)
    - "" -> "" (empty string stays empty)
    - "a..b" -> "a--b" (consecutive dots become consecutive hyphens -- acceptable)

    build_remote_session_cmd:
    - ("host", "myws", None) -> "ssh -t host \"tmux new-session -A -s myws\""
    - ("user@server", "dev.app", Some("~/src")) -> "ssh -t user@server \"cd ~/src && tmux new-session -A -s dev-app\""
    - Always includes -t flag (TTY allocation, SESS-04)
    - Always uses new-session -A -s (atomic attach-or-create)
    - Session name is sanitized before embedding in command
  </behavior>
  <implementation>
    1. Create src/session.rs with two public functions:
       - `pub fn sanitize_session_name(name: &str) -> String` -- Map dots and colons to hyphens, strip other non-alphanumeric/hyphen/underscore chars, trim leading/trailing hyphens.
       - `pub fn build_remote_session_cmd(host: &str, session_name: &str, dir: Option<&str>) -> String` -- Build the SSH command string. Calls sanitize_session_name internally. With dir: `ssh -t {host} "cd {dir} && tmux new-session -A -s {sanitized}"`. Without dir: `ssh -t {host} "tmux new-session -A -s {sanitized}"`.
    2. Add `pub mod session;` to src/lib.rs (after the existing `pub mod ssh;` line).
    3. Tests go in #[cfg(test)] mod tests inside src/session.rs (Rust convention -- tests beside code).
  </implementation>
</feature>
